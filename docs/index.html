01 / What you get

A plugin that updates itself.

01

Auto-updates from GitHub

Wires up yahnis-elsts/plugin-update-checker against your release tags. Push a tag, sites running the plugin see an update inside WordPress.

02

Release-ready Action

A bundled GitHub Actions workflow builds the plugin zip and attaches it to a release on every tag. Composer + npm steps included; no copy-paste.

03

Multi-block by default

Blocks live under src/<slug>/. Add another with --no-plugin. Server-rendered (render.php), Interactivity API on the interactive variant.

02 / Generated structure

The shape of a fresh plugin.

my-plugin/
├── my-plugin.php          // main plugin file
├── plugin.json            // updater manifest
├── readme.txt             // WP.org-style readme
├── composer.json
├── .github/
│   └── workflows/
│       └── main.yml      // build + release
├── src/
│   └── my-plugin/
│       ├── block.json
│       ├── edit.js
│       ├── render.php
│       └── view.js
└── build/                  // (generated)
my-plugin.php (excerpt)
<?php
/**
 * Plugin Name: My Plugin
 * Version:     0.1.0
 */

use YahnisElsts\PluginUpdateChecker\v5\PucFactory;

$updater = PucFactory::buildUpdateChecker(
    'https://github.com/you/my-plugin/',
    __FILE__,
    'my-plugin'
);

add_action( 'init', function() {
    register_block_type( __DIR__ . '/build/my-plugin' );
} );
